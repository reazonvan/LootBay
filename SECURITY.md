# Security Policy

## Supported Versions

Мы поддерживаем безопасность для следующих версий:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

Если вы обнаружили уязвимость безопасности, пожалуйста, **НЕ** создавайте публичный Issue.

Вместо этого:

1. **Отправьте email** на: security@lootbay.com
2. **Или создайте приватный Issue** с тегом `[SECURITY]`

### Что включить в отчет:

- Описание уязвимости
- Шаги для воспроизведения
- Возможное влияние
- Предложения по исправлению (если есть)

### Что мы делаем:

1. Подтверждаем получение отчета в течение 48 часов
2. Исследуем уязвимость
3. Разрабатываем исправление
4. Публикуем обновление
5. Благодарим вас за отчет

## Безопасность проекта

### Аутентификация
- JWT токены с ограниченным временем жизни
- Refresh токены для безопасного обновления
- Хеширование паролей с bcrypt

### API Безопасность
- Rate limiting для предотвращения атак
- CORS настройки
- Валидация входных данных
- SQL injection защита

### База данных
- Подготовленные запросы
- Минимальные права доступа
- Регулярные бэкапы

### Контейнеризация
- Непривилегированные контейнеры
- Сканирование образов на уязвимости
- Обновление базовых образов
