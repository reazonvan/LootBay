# Contributing to LootBay

Спасибо за интерес к проекту LootBay! Мы приветствуем вклад от сообщества.

## Как внести свой вклад

### 1. Форк и клонирование

1. Сделайте форк репозитория
2. Клонируйте ваш форк локально:
   ```bash
   git clone https://github.com/your-username/LootBay.git
   cd LootBay
   ```

### 2. Настройка окружения

1. Убедитесь, что у вас установлены:
   - Go 1.21+
   - Docker и Docker Compose
   - Node.js 18+ (для фронтенда)

2. Запустите проект локально:
   ```bash
   docker-compose up -d
   ```

### 3. Создание ветки

Создайте новую ветку для ваших изменений:
```bash
git checkout -b feature/your-feature-name
```

### 4. Внесение изменений

- Следуйте существующим соглашениям по кодированию
- Добавляйте комментарии к коду
- Пишите тесты для новой функциональности
- Обновляйте документацию при необходимости

### 5. Коммиты

Используйте понятные сообщения коммитов:
```bash
git commit -m "feat: add user role management"
git commit -m "fix: resolve authentication issue"
git commit -m "docs: update API documentation"
```

### 6. Push и Pull Request

1. Отправьте изменения в ваш форк:
   ```bash
   git push origin feature/your-feature-name
   ```

2. Создайте Pull Request с описанием изменений

## Стандарты кода

### Go
- Используйте `gofmt` для форматирования
- Следуйте [Effective Go](https://golang.org/doc/effective_go.html)
- Добавляйте комментарии к экспортируемым функциям

### TypeScript/React
- Используйте TypeScript строго
- Следуйте ESLint правилам
- Используйте функциональные компоненты с хуками

### База данных
- Создавайте миграции для изменений схемы
- Тестируйте миграции на тестовых данных

## Тестирование

Запустите тесты перед отправкой PR:
```bash
# Go тесты
go test ./...

# Фронтенд тесты
cd web
npm test
```

## Структура проекта

- `cmd/` - точки входа сервисов
- `internal/` - внутренняя логика приложения
- `web/` - Next.js фронтенд
- `migrations/` - SQL миграции
- `docker/` - Dockerfile'ы
- `docs/` - документация

## Вопросы и поддержка

Если у вас есть вопросы, создайте Issue в репозитории.

Спасибо за ваш вклад! 🚀
