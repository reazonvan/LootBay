# MCP Server для FunPay проекта

## Описание

MCP (Model Context Protocol) сервер предоставляет AI ассистенту доступ к функциональности вашего FunPay проекта через стандартизированный протокол.

## Возможности

MCP сервер предоставляет следующие инструменты:

### Файловые операции
- `list_dir` - Просмотр содержимого директорий
- `read_file` - Чтение файлов
- `write_file` - Запись в файлы
- `delete_file` - Удаление файлов
- `search_files` - Поиск файлов по содержимому

### Работа с базой данных
- `db_query` - Выполнение SQL запросов
- `db_execute` - Выполнение команд БД

### API операции
- `api_call` - Вызов API эндпоинтов
- `list_endpoints` - Список доступных эндпоинтов

### Системные операции
- `run_command` - Выполнение команд терминала
- `get_project_info` - Информация о проекте
- `deploy_service` - Деплой сервисов

## Установка и запуск

### 1. Запуск через Docker Compose

```bash
# Запуск всех сервисов включая MCP
docker-compose up -d

# Проверка статуса MCP сервера
curl http://localhost:8088/health
```

### 2. Запуск в режиме разработки

```bash
# Сборка MCP сервера
go build -o bin/mcp-server ./cmd/mcp-server

# Запуск
./bin/mcp-server
```

## Конфигурация

### Для Cursor IDE

1. Откройте настройки Cursor
2. Найдите раздел MCP (Model Context Protocol)
3. Добавьте конфигурацию:

```json
{
  "mcpServers": {
    "funpay-mcp": {
      "command": "ws://localhost:8088/mcp",
      "description": "MCP сервер для работы с FunPay проектом"
    }
  }
}
```

### Для других IDE

Скопируйте содержимое `mcp-config.json` в настройки вашего IDE.

## Использование

После подключения MCP сервера, AI ассистент сможет:

1. **Читать и анализировать код** - просматривать файлы проекта
2. **Создавать и редактировать файлы** - писать новый код
3. **Работать с базой данных** - выполнять запросы
4. **Вызывать API** - тестировать эндпоинты
5. **Выполнять команды** - запускать скрипты и команды
6. **Получать информацию о проекте** - анализировать архитектуру

## Безопасность

MCP сервер настроен с учетом безопасности:

- Доступ только к файлам в пределах проекта
- Валидация всех путей
- Ограниченные права на выполнение команд
- Изоляция в Docker контейнере

## Мониторинг

MCP сервер доступен по адресу:
- WebSocket: `ws://localhost:8088/mcp`
- Health check: `http://localhost:8088/health`

## Устранение неполадок

### MCP сервер не запускается
1. Проверьте логи: `docker-compose logs mcp-server`
2. Убедитесь что порт 8088 свободен
3. Проверьте права доступа к файлам проекта

### Соединение не устанавливается
1. Проверьте что сервер запущен: `curl http://localhost:8088/health`
2. Убедитесь что WebSocket endpoint доступен
3. Проверьте конфигурацию в IDE

### Ошибки доступа к файлам
1. Проверьте что пути находятся в пределах проекта
2. Убедитесь что файлы существуют
3. Проверьте права доступа

## Разработка

### Добавление новых инструментов

1. Добавьте функцию в `cmd/mcp-server/main.go`
2. Зарегистрируйте инструмент в `registerTools()`
3. Добавьте описание в `handleToolsList()`

### Тестирование

```bash
# Тест WebSocket соединения
wscat -c ws://localhost:8088/mcp

# Тест HTTP endpoints
curl http://localhost:8088/health
```
